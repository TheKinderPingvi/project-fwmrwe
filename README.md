## FLY WITHOUT ME (RAILWAY EDITION)
Целью данного проекта является разработка системы автономной посадки БПЛА на несущий трос, автоматизирующей мониторинг и обслуживание контактной сети, обеспечивая безопасное приземление дрона на трос и возвращение его в домашнюю точку <br> Ожидаемый продуктовый результат - система автономной посадки “Fly Without Me”, решение которой включает в себя использование лидара для точного позиционирования, а также расчёт маршрута полёта и веб-интерфейс для управления

## Описание аппаратной части

Предполагается, что на дроне установлены

- Лазер LiDAR (в сборке установлены технические параметры Velabit Velodyne LiDAR  <br> ([технические характеристики](https://agmtrd.com/product/velabit)) , позиционные параметры установки относительно дрона <br> (xyz="0 0 -0.02" rpy="1.571 1.571 0"))

- Две видеокамеры (в сборке установлены параметры для обеих камер: Разрешение камеры 320x240, формат RGB, позиционные параметры установки относительно дрона <br> (xyz="0 0.06 0.11" rpy="0 0.2618 1.571"); (xyz="0 -0.06 0.11" rpy="0 0.2618 4.713"))
  


## Описание программной части

Описание программной части: в чём заключается решение и как оно работает

### Решение состоит из двух блоков :
1. Блок веб-интерфейса 
   + Программа запуска веб-интерфейса (app.py)
   + файл index.html - структура страницы
   + файл script.js - клиентская логика (подключение, отправка команд, приём данных)
   + файл style.css - оформление внешнего вида веб-интерфейса

2. Программный блок
   - config.py - параметры миссии (зона сканирования, размеры объектов, параметры дрона и др.)
   - fwm_rwe.py - основная программа автономного полёта
   - homecoming.py - программа возвращения в точку старта 
   - *+ TESTWIREFOLLOWER.py - программа имитации движения вдоль троса (после сцепки)

### Блок веб-интерфейса

![Картинка Веб-Интерфейса](https://github.com/TheKinderPingvi/project-fwmrwe/raw/main/newroutes/web-interface1.1.png)

> **Краткое описание пользовательского интерфейса:**

>**При использовании системы оператор видит перед собой веб-интерфейс с большим оконом изображения передней камеры БПЛА. <br> Прямо под окном расположены кнопки управления ("Запуск", "Экстренный возврат", "Пауза"). <br> Справа расположена панель с изображением задней камеры дрона, сгенерированным маршрутом миссии и окном логов (сообщений)**

<br>

Концептуальное описание блока:

#### Фаза 1. Запуск сервера

- При запуске app.py создаётся веб-приложение: Flask (веб-сервер на Python) с Socket.IO обеспечивает мгновенный обмен данными между интерфейсом и сервером

- Активируется обработка через eventlet для одновременной работы с несколькими соединениями

#### Фаза 2. Подключение к ROS

Запускается поток ROSListener:

   - Подключается к ROS как нода web_interface

   - Подписывается на топик /rosout_agg (логирование ROS)

   - Выводятся сообщения только от ноды /autonomous_flight

#### Фаза 3. Работа веб-интерфейса

- При загрузке страницы выводится HTML-интерфейс (index.html)

- После генерации изображение маршрута выводится в веб-интерфейс (scan_route_recent.png)

#### Фаза 4. Управление процессами дрона

При нажатии кнопки "Запуск" запускается событие start_mission:

- Запускается скрипт fwm_rwe.py (автономный полёт) как отдельный процесс

- Вывод процесса передаётся в веб-интерфейс в реальном времени

При нажатии кнопки "Экстренный возврат" запускается событие emergency_home:

- Останавливаются все активные процессы

- Запускается скрипт homecoming.py (возврат на точку старта)

### Программный блок
> **Краткое описание последовательности выполнения миссии:** <br>

>**Взлёт → Полёт к начальной точке зоны обследования → Первый проход над зоной (поиск консолей) + корреция высоты облёта → Второй проход на скоректированной высоте (обнаружение тросов) → Построение маршрута с обходом препятствий → Полёт вдоль верхнего троса (с коррекцией по лидару и облётом консолей) → Полёт вдоль нижнего троса (с учётом пересечений тросов и облётом консолей) → Возврат в точку старта и посадка**

<br>

Концептуальное описание блока:

#### Фаза 1. Инициализация ROS-ноды

Программа запускается как нода autonomous_flight в ROS

Создаёт прокси для сервисов Clover Platform:

- get_telemetry (получение телеметрии)

- navigate (управление движением)

- land (посадка)

#### Фаза 2. Настройка системы координат (TF)

- Создаёт статические преобразования координат для датчиков (положение датчиков относительно дрона):

- Публикует преобразования через StaticTransformBroadcaster

#### Фаза 3. Обработка сенсорных данных

Лидар (Velodyne):

- Подписка на топик /velodyne_points

- Преобразование облака точек в систему aruco_map



#### Фаза 4. Сканирование зоны (первый проход, поиск консолей)

 - Дрон движется вдоль заданной зоны с включённым лидаром

 - Собирается трёхмерное облако точек

 - Строится гистограмма высот, ищутся пики в районе ожидаемой высоты консолей (extract_cables_and_consoles)

 - Близко расположенные консоли объединяются в одну зону (merge_console_zones)



#### Фаза 5. Сканирование зоны (второй проход, поиск тросов)

- На основе максимальной высоты консолей вычисляется безопасная высота второго прохода (в сборке указано значение на 0,5 м выше консолей)

- Дрон поднимается на эту высоту и выполняет обратный проход, снова собирая лидарные точки

- Собранные точки фильтруются для отсеивания шумов и уточнения маршрута

- Повторно вызывается extract_cables_and_consoles, выделяются два самых значимых пика гистограммы: верхний и нижний несущий трос

- Для каждого троса вычисляются точки маршрута и рабочая высота полёта 

#### Фаза 6 Построение маршрута с обходом препятствий 

- По опорным точкам создаётся сетка путевых точек (build_waypoints)

- Для верхнего троса маршрут пропускается через зоны консолей: в местах наложения дрон поднимается на безопасную высоту, горизонтально пересекает зону и спускается (build_avoidance_waypoints)

- Для нижнего троса дополнительно учитывается точка пересечения тросов и формируется зона облёта (find_crossing_point)

- Маршрут сохраняется в виде графика (plot_scan_data) в папку newroutes/scan_route.png; последний маршрут копируется в newroutes/scan_route_recent.png для веб-интерфейса

#### Фаза 7 Выполнение полёта по верхнему тросу

- Дрон перемещается к началу верхнего троса (сначала на безопасной высоте, затем на рабочую)

- Перед "сцепкой" выполняется коррекция позиции по данным лидара (correct_position)

- Последовательно проходятся сегменты маршрута:
   - После "сцепки" с тросом запускается скрипт TESTWIREFOLLOWER.py, который ведёт дрон вдоль троса
   - При входе в зону обхода дрон "отцепляется" от троса, облетает препятствие по рассчитанным путевым точкам, затем восстанавливает "сцепку" с тросом и продолжает движение
   
- Каждая смена режима логируется и выводится в веб-интерфейс

- После маршрута верхнего троса дрон "отцепляется" от троса, взлетает на безопасную высоту и перемещается к началу маршрута нижнего троса

#### Фаза 8 Выполнение полёта по нижнему тросу

- Движение выполняется аналогично полёту предыдущей фазы

- Для зоны пересечения контактных подвесок предусмотрен особый манёвр (если позволяет свободное пространство (в сборке указано значение ±0.3 м от диагонали габаритов аппарата по оси троса)):
   - Выполняется челночный проход (движение вперёд‑назад) с возвратом к началу следующей зоны

   - Если места недостаточно, зона пересечения ещё на этапе планирования объединяется с ближайшей консольной зоной для облёта как единого препятствия

#### Фаза 9 Послепосадочная последовательность (движение вдоль тросов и возврат в домашнюю точку)

Запуск послепосадочных процессов:

- TESTWIREFOLLOWER.py (имитация движения после сцепки вдоль троса)

- homecoming.py (возврат в точку взлёта)

#### Взаимодействие с веб-интерфейсом

- Логирование в формате [FWM_RWE] сообщение

- Передача в веб-интерфейс маршрута scan_route_recent.png через Flask

- Запуск/остановка программы через Socket.IO (управляется из app.py)

## Стек технологий
- Frontend : HTML , CSS
- Backend : JavaScript , Python
- Библиотеки :
  - ROS-библиотеки : 
      - Основные ROS-клиенты и инструменты : <br> ros-noetic-rospy (Python-клиент для ROS) , python3-rosdep (инструмент управления системными зависимостями) , ros-noetic-rosgraph-msgs (сообщения для внутренней диагностики ROS)
      - Системы координат и преобразования (TF) : <br> ros-noetic-tf2-ros (основной инструмент TF2 для ROS)  , python3-tf2-msgs (Python-интерфейс для сообщений TF2) , ros-noetic-tf2-sensor-msgs (преобразования для данных сенсоров)
      - ROS-сообщения для различных данных : <br> ros-noetic-sensor-msgs (форматы данных сенсоров: камеры, IMU, лазеры) , ros-noetic-geometry-msgs (геометрические примитивы: точки, векторы, Pose) , ros-noetic-nav-msgs (сообщения для навигации) , ros-noetic-std-srvs (стандартные сервисные запросы/ответы)
      - Взаимодействия с ROS через веб-интерфейс : <br> ros-noetic-rosbridge-server (WebSocket для доступа к ROS из браузера) , ros-noetic-rosauth (аутентификация для rosbridge-server)
      - Библиотеки COEX Clover : <br> ros-noetic-clover (пакет для работы с квадрокоптером Clover)

  - Python-библиотеки : numpy (работа с массивами данных , математические операции для анализа распределения точек лидара, фильтрация данных для выделения точек в зоне тросов , генерация контрольных точек) , matplotlib (построение маршрута с сохранением в PNG), eventlet (асинхронные сетевые операции в веб-интерфейсе) , flask (веб-фреймворк для интерфейса управления) , flask-socketio (интеграция WebSocket для Flask), scipy (алгоритм поиска пиков при анализе облака точек)
- Инструменты : Gazebo COEX Clover , RViz , LiDAR
